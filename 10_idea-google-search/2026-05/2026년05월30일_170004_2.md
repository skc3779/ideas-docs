# LabPilot Vertical AI - 과학 연구 실험/논문 워크플로 자동화 플랫폼

## 1. 제안 범위 및 카테고리
- 카테고리: 산업 특화 AI (Vertical AI)

## 2. 문제 정의
연구 조직은 데이터 수집, 실험 설계, 문헌 검토, 결과 재현 과정이 분절되어 있다.
- 실험 기록 포맷이 제각각이라 재현성 저하
- 논문/리포트 작성 이전의 데이터 정제에 과도한 시간 소요
- 연구 인력의 도구 숙련도 편차로 생산성 차이 확대

## 3. 타겟 사용자
- 대학/출연연 연구실
- 딥테크 스타트업의 R&D 팀
- 과학 교육기관의 프로젝트형 수업 운영자

## 4. 핵심 기능
- 실험 계획서 자동 생성: 목적/가설/변수 입력 시 표준 프로토콜 제안
- 문헌 근거 추적: 주장-근거-인용 자동 연결 그래프 제공
- 실험 로그 표준화: 장비 로그와 수기 기록을 통합 저장
- 결과 검증 에이전트: 이상치, 누락 데이터, 재현성 경고 자동 표시
- 초안 자동화: 연구노트 -> 기술보고서/발표자료 초안 변환

## 5. MVP 범위
- 과학 분야 2개 트랙(바이오, 재료) 템플릿 제공
- 실험 로그 업로드 + 기본 통계/시각화
- 문헌 100편 내 검색/근거 링크 기능
- 보고서 초안 생성(국문/영문)

## 6. 수익 모델
- 기관 라이선스(연 단위 사용자 슬롯)
- 연구실 단위 구독(소규모 플랜)
- API 과금(외부 LIMS/ELN 연동)

## 7. Go-to-Market
- 대학/공공 연구기관과 공동 파일럿
- 연구 재현성/문서화 품질 개선 지표를 KPI로 제시
- 학회/연구지원단 대상 데모 세션으로 확산

## 8. 개발/운영 로드맵
- 0~2개월: 데이터 스키마/로그 파서/문헌 인덱서 구축
- 3~4개월: 2개 연구실 파일럿, 실험 템플릿 정교화
- 5~6개월: 검증 에이전트 고도화, 다기관 협업 기능 추가
- 7개월+: 규제/심사 대응 문서 자동화 기능 확장

## 9. 리스크 및 대응
- 리스크: 연구 데이터 보안 및 IP 유출 우려
- 대응: 프로젝트별 격리 저장소, 세부 권한, 온프렘 옵션 제공

- 리스크: 연구 분야별 워크플로 차이
- 대응: 공통 코어 + 도메인별 플러그인 구조 채택

## 10. 실현 가능성 점수 및 근거
- 점수: 4/5
- 근거: 과학 AI 투자 및 프로젝트 수요는 강하나, 기관별 데이터 표준 정합 작업이 초기 진입 장벽
