# [제안 3 상세 실행 방안] 고유가 시대 맞춤형 '가성비 국제선' AI 트래커 및 여행 플래너

## 1. 서비스 개요 및 타겟 고객
- **서비스명 (가칭)**: BudgetFly AI
- **타겟 고객**: 해외여행을 계획 중이나 고유가/고물가로 인해 항공권 및 여행 경비에 부담을 느끼는 20~40대 여행객. 목적지보다는 '예산'에 맞춰 여행지를 결정할 의향이 있는 유연한 여행자.

## 2. 핵심 기능
1. **예산 역산형 목적지 추천**: 사용자가 "총예산 100만 원, 3박 4일"을 입력하면, 현재 실시간 항공권 가격과 현지 물가(숙박비, 식비 평균)를 계산하여 예산 내에서 다녀올 수 있는 최적의 해외 도시들을 리스트업.
2. **AI 항공권 가격 트래커**: 관심 있는 노선의 가격 변동을 AI가 24시간 모니터링하다가, 과거 데이터 대비 '초특가' 구간에 진입하거나 항공사 프로모션이 뜨면 카카오톡/앱 푸시로 즉각 알림.
3. **가성비 최적화 AI 일정표**: 챗봇 형태의 UI를 통해 무료 관광지, 현지인 가성비 맛집 위주로 동선을 짜주는 스마트 플래너 기능.

## 3. 수익 모델 (BM)
- **제휴 수수료 (Affiliate)**: 스카이스캐너, 아고다, 클룩 등 OTA 플랫폼으로 트래픽을 넘겨주고 발생하는 예약 건당 수수료(CPA) 수취.
- **프리미엄 알림 서비스**: 월 2,900원 등의 소액 결제로 남들보다 10분 먼저 특가 알림을 받을 수 있는 구독 모델.

## 4. 기술 스택 및 개발 단계
- **프론트엔드**: Next.js, Tailwind CSS (웹 기반 빠른 출시 후 앱 전환)
- **백엔드**: Node.js (Express), MongoDB
- **외부 API**: Skyscanner API, Amadeus API (항공권 데이터), OpenAI API (일정 생성)
- **1단계 (MVP)**: 예산 입력 시 갈 수 있는 국가 추천 및 항공권 최저가 링크 제공.
- **2단계**: 실시간 가격 트래킹 및 푸시 알림 시스템 구축.

## 5. 리스크 및 대응 방안
- **항공권 API 비용 및 딜레이**: 실시간 항공권 데이터를 지속적으로 호출하면 API 비용이 급증할 수 있음. 자체 캐싱(Caching) 시스템을 구축하여 동일 검색어에 대한 중복 호출을 최소화하고, 배치(Batch) 작업으로 주요 노선 데이터를 주기적으로 업데이트하는 방식 채택.
