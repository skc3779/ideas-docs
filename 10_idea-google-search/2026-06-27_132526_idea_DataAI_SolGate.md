# [아이디어 상세 실행 문서] SolGate (솔게이트)

## 1. 아이디어명
- **SolGate (솔게이트)**: 장기 가동(Long-Horizon) 에이전트용 실시간 툴체인 감사 및 장애 제어 MLOps

---

## 2. 제안 범위 및 카테고리
- **대범위**: 데이터 및 인공지능 (Data & AI)
- **중분류/소분류**: MLOps 및 생성형 AI 응용 (LLM App)

---

## 3. 해결 문제와 사용자 페르소나

### 해결하고자 하는 문제 (Pain Point)
1. **장기 가동 에이전트의 모니터링 난제**: OpenAI의 GPT-5.6 Sol 및 신규 에이전트 아키텍처는 단발성 질답을 넘어 한 번의 지시로 수십 분에서 수 시간 동안 스스로 터미널 명령을 치고, 브라우저를 서핑하며, 파일을 변경하는 '장기 가동(Long-Horizon) 자율 에이전트' 시대를 열었습니다. 하지만, 에이전트가 가동되는 긴 시간 동안 백그라운드에서 어떤 잘못된 도구(Tool Calling)를 수행하는지 한눈에 보거나 중단시킬 감사 인프라가 부재합니다.
2. **에러 루핑에 따른 비용 폭발과 행(Hang) 현상**: 에이전트가 코딩이나 보안 문제를 풀다가 에러에 부딪히면 무의미한 API 호출 루프를 반복하여 수백 달러의 API 토큰 요금이 순식간에 청구될 수 있습니다.
3. **위험 행동 사전 제어의 부재**: 최상위 모델 'Sol'은 사이버 보안, 생물학 등의 영역에서 전문가 수준의 자율 실행력을 보입니다. 기업 내부 망에서 실행 시 혹여나 자사 인프라에 심각한 피해를 주는 고위험 명령어(시스템 파괴, 고위험 외부 패키지 다운로드)를 사전에 가드레일 할 장치가 부족합니다.

### 사용자 페르소나
- **신동아 (31세, AI 에이전트 프로덕트 매니저)**: 사내에 고객 가입 프로세스를 자율적으로 자동 처리하는 GPT-5.6 기반 에이전트 워커를 도입하려 합니다. 에이전트가 중간에 예외적 상황을 마주했을 때 관리자에게 알림을 주고 승인을 받는 체계를 통합 대시보드로 통제하고 싶어 합니다.
- **오기민 (39세, 인프라 및 플랫폼 엔지니어)**: 개발용 터미널 내에서 자율 작동하며 코딩하는 AI 에이전트들에게 자사 서버 인프라에 접근 권한을 줄 때, 위험한 쓰기/삭제 명령어들을 사전에 걸러내고, 비상시 작동을 강제 종료시킬 수 있는 보안 가로채기(Proxy Gate) 레이어를 찾고 있습니다.

## 4. 핵심 기능 및 차별점

### 핵심 기능
1. **Live Agent Trace Graph (실시간 에이전트 경로 시각화)**
   - 에이전트가 가동을 시작한 후 독립적으로 수행하는 서브 태스크, 툴 호출 히스토리, 상태 변화를 트리 구조의 인터랙티브 그래프로 실시간 렌더링해 줍니다.
2. **Dynamic Human-in-the-Loop Gate (동적 개입 및 가드레일)**
   - 고위험 액션(예: 파일 영구 삭제, 외부 네트워크 아웃바운드 전송, 핵심 데이터베이스 수정 등) 감지 시, 에이전트 런타임을 자동으로 일시 정지(Pause) 시키고, 인간 매니저의 Slack/웹 확인 요청을 띄워 '승인(Approve)', '대안 제안(Modify)', '반려(Reject)'를 입력받은 후 실행을 이어나가게 합니다.
3. **Infinite Loop Breaker & Rollback Points (오류 무한루프 자동 차단)**
   - 에이전트가 동일한 에러 로그나 툴 호출을 5회 이상 유의미한 진전 없이 반복 시, 과도한 토큰 소모를 방지하기 위해 런타임을 차단하고 안전한 이전 메모리 상태(Rollback State)로 시스템 컨텍스트를 되돌려 복원 처방 프롬프트를 주입합니다.

### 차별점
- **GPT-5.6 Sol 전용 통합 어댑터 내장**: 최신 GPT-5.6 Sol의 세부 하위 모델 구조('테라', '루나' 포함)와 터미널 벤치마크 2.1 보안 사양을 완벽히 흡수하여 최적의 파라미터 매칭을 이뤄냅니다.
- **초경량 프록시 아키텍처**: 에이전트가 사용하는 라이브러리(LangChain, CrewAI, Autogen 등)의 엔드포인트 코드 단 두 줄만 변경하여 SolGate 보안 프록시로 우회시킬 수 있어 도입이 매우 간편합니다.

## 5. 실행 계획 (MVP -> 확장)

### MVP 단계
- **에이전트 로깅 및 실시간 일시 정지 미들웨어**: LangChain 기반 에이전트 프로젝트에 붙일 수 있는 가벼운 SDK와, 호출 경로를 모니터링하면서 특정 위험 명령 발생 시 실행을 멈추고 콘솔이나 웹상에서 동의를 얻는 대화형 게이트웨이 라이브러리 개발.

### 확장 단계
- **엔터프라이즈 에이전트 SOC (보안 관제 센터)**: 수백 명의 사내 직원이 사용하는 수천 마리의 AI 에이전트의 구동 세션 전체를 실시간 모니터링하고 토큰 과금 한도 통제, 에이전트 자격 증명(Credential) 관리를 일괄 처리하는 기업 전용 웹 대시보드로 서비스 고도화.

## 8. 실현 가능성 점수 및 근거
- **실현 가능성 점수**: **4.6 / 5.0** (매우 높음)
- **근거**:
  - OpenAI가 GPT-5.6 Sol 및 에이전트 중심의 노동 혁신 연구 자료(How agents are transforming work)를 공식 발표하면서, 업계의 중심은 Chat 기반에서 자율 Agent 기반으로 완전하게 이동하기 시작했습니다.
  - 단발성 대화 기록을 로깅하던 기존 인프라(LangSmith, Helicone 등)로는 장시간 자율 운용되는 에이전트의 위협 요소나 중간 도구 호출 흐름을 가드레일하고 승인할 수 없으므로, 에이전트 중심의 전용 MLOps 감사 레이어는 즉각적인 신규 수요가 보장되는 매력적인 솔루션입니다.
