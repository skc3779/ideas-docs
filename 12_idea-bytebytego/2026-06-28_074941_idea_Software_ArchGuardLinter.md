# [아이디어 상세 실행 문서] ArchGuard Linter (아키가드 린터)

## 1. 아이디어명
- **ArchGuard Linter (아키가드 린터)**: 마이크로서비스 아키텍처 안티 패턴 정적 분석 도구

---

## 2. 제안 범위 및 카테고리
- **대범위**: 소프트웨어 (Software)
- **중분류/소분류**: DevTools 및 Cloud Infrastructure

---

## 3. 해결 문제와 사용자 페르소나

### 해결하고자 하는 문제 (Pain Point)
1. 많은 기업이 모놀리식 시스템의 한계를 느껴 마이크로서비스(MSA)로 전환하지만, 잘못된 분리(Tight Coupling), 과도한 서비스 간 호출(Chatty Services) 등의 안티 패턴으로 인해 오히려 장애가 빈발하고 비용이 상승합니다.
2. 각 개발팀은 개별 서비스의 코드 품질은 신경 쓰지만, 전체 서비스 간의 상호작용과 아키텍처 부채를 리뷰하고 경고해 주는 자동화된 도구가 없습니다.

### 사용자 페르소나
- **플랫폼 엔지니어 및 DevOps 팀장**: 수백 개의 마이크로서비스가 얽힌 사내 시스템의 아키텍처 무결성을 유지하고 싶으며, 새로운 배포가 전체 시스템의 안티 패턴을 심화시키지 않도록 강제(Guardrail)하고 싶습니다.

---

## 4. 핵심 기능 및 차별점

### 핵심 기능
1. **아키텍처 토폴로지 분석**: 소스 코드(API 호출 구문)와 인프라 구성 파일(Terraform, Kubernetes YAML)을 스캔하여 전체 서비스 토폴로지를 자동으로 시각화합니다.
2. **안티 패턴 탐지 룰 엔진**: ByteByteGo 등에서 정의한 '공유 데이터베이스 안티 패턴', '분산 트랜잭션 락' 등의 룰셋을 기반으로 설계 결함을 탐지하여 CI/CD 파이프라인에서 경고(Warning)를 발생시킵니다.
3. **LLM 기반 리팩토링 가이드 제안**: "서비스 A와 B가 너무 빈번하게 통신합니다. GraphQL이나 이벤트 드리븐 아키텍처(Kafka 연동)로의 변경을 추천합니다"와 같이 구체적인 개선안을 제시합니다.

### 차별점
- 단순한 코드 문법 검사(SonarQube)나 보안 취약점 스캔을 넘어, **'분산 시스템 아키텍처 설계' 자체의 결함을 찾아내는 하이레벨 Linter**입니다.

---

## 5. 실현 가능성 점수 및 근거
- **실현 가능성 점수**: **4.3 / 5.0** (높음)
- **근거**: OpenTelemetry 분산 트레이싱 데이터와 LLM의 코드/설정 파일 이해력을 결합하면 논리적인 결합도 추론이 충분히 가능하며, B2B SaaS로서 구독 가치가 뚜렷합니다.
