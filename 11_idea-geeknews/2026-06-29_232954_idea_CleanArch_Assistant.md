# 1. 아이디어명
CleanArch Assistant (클린 아키텍처 자동 리팩토링 AI 툴)

# 2. 제안 범위 및 카테고리
소프트웨어 (Software) - 개발 도구 (DevTools)

# 3. 해결 문제와 사용자 페르소나
- **해결 문제**: 외부 라이브러리 및 프레임워크와 비즈니스 로직이 강하게 결합된 레거시 스파게티 코드를 분리하는 작업은 시간이 많이 들고 오류 발생 확률이 높음.
- **사용자 페르소나**: 유지보수가 어려운 시스템을 관리하고 있는 시니어 개발자, 기술 부채를 해결해야 하는 스타트업 CTO 및 백엔드 개발팀.

# 4. 핵심 기능/차별점
- **의존성 시각화 및 분리 추천**: 코드베이스를 스캔하여 로버트 C. 마틴의 클린 아키텍처 기준에 맞게 계층간 의존성을 분석하고 위반 사항을 시각화.
- **AI 리팩토링 봇**: 외부 문제(데이터베이스, UI, 프레임워크)를 분리하여 도메인 비즈니스 로직을 보호하는 형태의 코드 리팩토링 방안을 제시하고 PR(Pull Request) 자동 생성.
- **테스트 코드 자동화**: 계층별로 독립적인 단위 테스트(Unit Test)를 자동 생성하여 시스템 검증 가능성 향상.

# 5. 실행 계획 (MVP -> 확장)
- **MVP 단계**: GitHub/GitLab 연동 후 특정 언어(예: TypeScript/Python) 기반의 단일 프로젝트 리포지토리를 스캔하여 클린 아키텍처 위반 사항을 레포트로 생성하는 SaaS.
- **확장 단계**: 개발 IDE(VSCode, IntelliJ) 플러그인 제공, AI 기반 자동 리팩토링 코드 삽입 기능 및 의존성 주입(DI) 컨테이너 자동 구성 기능 추가.

# 6. 수익 모델
- **Freemium 모델**: 개인 개발자 및 오픈소스 프로젝트용 무료 분석 보고서 제공.
- **B2B 구독(SaaS)**: 기업 팀 단위 월간/연간 구독 (코드 라인 수 또는 리포지토리 수 비례). 
- **엔터프라이즈 도입**: 온프레미스 기반 로컬 리팩토링 분석 도구 라이선스 판매.

# 7. 일정/리소스 계획
- **1개월**: LLM(GPT-4 등)을 활용한 코드 파싱 및 분석 프롬프트 설계, MVP 백엔드 개발.
- **2개월**: GitHub App 연동 및 레포트 생성 자동화, 베타 테스트 진행.
- **3개월**: 피드백 반영 및 IDE 플러그인 초기 버전 개발/런칭.
- **리소스**: 백엔드 개발자 1명, 프롬프트 엔지니어 1명, 프론트엔드/익스텐션 개발자 1명.
